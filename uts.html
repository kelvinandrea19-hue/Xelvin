Tentang Saya

Pendidikan

Saat ini menempuh pendidikan di bidang Informatika. Mata kuliah Organisasi Komputer menjadi fondasi penting dalam memahami cara kerja hardware dan software secara terintegrasi.

Minat & Ketertarikan

Tertarik pada bidang arsitektur komputer, sistem tertanam (embedded systems), serta pengembangan perangkat lunak tingkat rendah (low-level programming) yang berinteraksi langsung dengan hardware.

Tujuan Akademik

Memahami secara mendalam bagaimana komputer bekerja dari perspektif hardware — mulai dari siklus instruksi CPU, manajemen memori, hingga komunikasi antar komponen melalui modul I/O.

Mata Kuliah Ini

Organisasi & Arsitektur Komputer memberikan pemahaman tentang struktur internal komputer, perbedaan antara organisasi dan arsitektur, serta cara CPU, memory, dan I/O saling bekerja sama.

Organisasi Komputer

Organisasi komputer mengacu pada unit operasional dan interkoneksi antar komponen yang mewujudkan spesifikasi arsitektur. Ini mencakup sinyal kontrol, antarmuka komputer-periferal, dan teknologi memori.

🧠 CPU

Mengontrol operasi komputer dan melakukan pemrosesan data. Terdiri dari ALU (Arithmetic Logic Unit), Control Unit, dan Register. CPU mengambil instruksi dari memori, mengeksekusi instruksi tersebut, dan menyimpan hasilnya.

💾 Main Memory

Menyimpan data dan instruksi yang sedang diproses CPU. Bersifat volatile (hilang saat listrik mati). Kapasitas lebih besar dari cache namun lebih lambat. Dihubungkan ke CPU via memory bus.

🔌 I/O Module

Menjembatani komunikasi antara komputer dengan perangkat eksternal (keyboard, monitor, disk, printer). Mengelola transfer data antara perangkat I/O dengan memori atau CPU.

🚌 System Bus

Media komunikasi yang menghubungkan CPU, memori, dan modul I/O. Terdiri dari Data Bus (transfer data), Address Bus (alamat memori), dan Control Bus (sinyal kontrol).

Perbedaan Konsep

Arsitektur vs Organisasi

🏛 Arsitektur Komputer

Atribut yang terlihat oleh programmer — set instruksi, jumlah bit, mekanisme I/O, teknik pengalamatan. Contoh: apakah ada instruksi perkalian dalam CPU. Bersifat abstrak/logis.

⚙️ Organisasi Komputer

Unit operasional dan interkoneksinya yang mewujudkan arsitektur — sinyal kontrol, antarmuka periferal, teknologi memori. Contoh: bagaimana instruksi perkalian diimplementasikan. Bersifat konkret/fisik.

Memory Hierarchy

RAM vs ROM

⚡ RAM — Volatile

  • Membutuhkan daya listrik untuk mempertahankan data
  • Data hilang saat listrik dimatikan
  • Digunakan untuk menyimpan data aktif & program yang sedang berjalan
  • Read & Write (baca & tulis)
  • Kecepatan tinggi, kapasitas besar

🔒 ROM — Non-Volatile

  • Data tersimpan secara permanen tanpa daya
  • Data tidak hilang saat listrik dimatikan
  • Digunakan untuk firmware & BIOS
  • Read Only (hanya baca) — ditulis saat pembuatan chip
  • Data tersimpan dalam struktur fisik chip

Modul I/O

Modul I/O (Input/Output) adalah komponen yang menghubungkan CPU dan memori dengan perangkat eksternal. Ia bertindak sebagai perantara yang mengelola perbedaan kecepatan, format data, dan protokol komunikasi.

// Posisi Modul I/O dalam Sistem
CPU
←→ System Bus ←→
I/O Module
←→
Keyboard / Mouse
Monitor / Printer
Hard Disk / USB

🔄 Sinkronisasi Kecepatan

CPU bekerja jauh lebih cepat daripada perangkat I/O. Modul I/O menggunakan buffer dan mekanisme handshaking untuk menjembatani perbedaan kecepatan ini, sehingga CPU tidak perlu menunggu perangkat lambat.

📡 Konversi Format Data

Mengubah format data antara representasi internal CPU (biner paralel) dan format yang digunakan perangkat eksternal (mis. serial untuk USB, format karakter untuk printer).

🚦 Deteksi Error

Modul I/O dapat mendeteksi error pada data yang diterima dari perangkat eksternal dan melaporkannya ke CPU, menjamin integritas data dalam transfer.

⚡ DMA — Direct Memory Access

Teknik di mana modul I/O dapat mentransfer data langsung ke/dari memori tanpa keterlibatan CPU setiap saat, membebaskan CPU untuk tugas lain dan meningkatkan efisiensi sistem secara keseluruhan.

🔔 Interrupt Handling

Modul I/O mengirim sinyal interrupt ke CPU saat perangkat selesai melakukan tugasnya, sehingga CPU tidak perlu terus-menerus memeriksa (polling) status perangkat.

🛡️ Isolasi & Proteksi

Memisahkan sistem bus internal (high-speed) dari perangkat eksternal yang lebih lambat dan rentan, melindungi CPU dari gangguan langsung perangkat eksternal.

Pertanyaan & Jawaban

1 Apa yang melatarbelakangi perbedaan antara Organisasi Komputer dan Arsitektur Komputer?

Perbedaan antara Arsitektur Komputer dan Organisasi Komputer dilatarbelakangi oleh dua sudut pandang yang berbeda dalam memandang sebuah sistem komputer.


Arsitektur Komputer merujuk pada atribut-atribut yang secara langsung terlihat dan relevan bagi programmer — meliputi set instruksi (ISA), jumlah bit yang digunakan untuk merepresentasikan data, mekanisme I/O, dan teknik pengalamatan memori. Arsitektur adalah "apa yang bisa dilakukan" sebuah komputer dari sisi logis/abstrak.

"Computer architecture refers to those attributes of a system visible to a programmer or those attributes that have a direct impact on the logical execution of a program." — William Stallings, Computer Organization and Architecture, 10th Edition, 2016. → Referensi: Stallings, W. (2016). Computer Organization and Architecture, 10th Ed. Pearson.

Organisasi Komputer merujuk pada unit operasional dan interkoneksi antar komponen yang mewujudkan spesifikasi arsitektur — meliputi sinyal kontrol, antarmuka periferal, dan teknologi memori yang digunakan. Organisasi adalah "bagaimana cara melakukannya" secara fisik/konkret.


Latar belakang perbedaan ini adalah kebutuhan praktis: banyak produsen menawarkan sekelompok model komputer dengan arsitektur yang sama tetapi organisasi yang berbeda-beda. Misalnya, processor Intel 64-bit memiliki arsitektur (ISA x86-64) yang sama, tetapi organisasi internal (implementasi fisik chip, jumlah cache, kecepatan bus) berbeda di setiap generasi produk.

→ Referensi: Patterson, D.A. & Hennessy, J.L. (2017). Computer Organization and Design RISC-V Edition. Morgan Kaufmann.

CPU (Central Processing Unit): Otak komputer. Bertugas mengambil instruksi dari memori (fetch), menerjemahkan instruksi (decode), dan mengeksekusi instruksi (execute). CPU terdiri dari ALU, Control Unit, dan Register.


Main Memory: Menyimpan program (kode instruksi) dan data yang sedang aktif diproses CPU. Bersifat volatile — data hilang saat komputer dimatikan. Kapasitas jauh lebih besar dari cache namun lebih lambat.


I/O Module: Menghubungkan CPU dan memori dengan perangkat eksternal. Menangani perbedaan kecepatan, format data, dan protokol antar komponen. Mendukung mekanisme interrupt dan DMA.


System Bus: Jalur komunikasi bersama yang menghubungkan CPU, memori, dan I/O. Terdiri dari Data Bus (membawa data), Address Bus (membawa alamat), dan Control Bus (membawa sinyal kontrol).

→ Referensi: Stallings, W. (2016). Computer Organization and Architecture, 10th Ed., Ch.1. Pearson.

Control Unit (CU): Mengambil instruksi dari memori, men-decode instruksi, dan menghasilkan sinyal kontrol untuk mengarahkan komponen lain. Bertindak sebagai "konduktor orkestra" CPU.


ALU (Arithmetic Logic Unit): Melaksanakan semua komputasi matematis dan logika. Operasi aritmatika: penjumlahan (+), pengurangan (-), perkalian (×), pembagian (÷). Operasi logika: AND, OR, NOT, XOR, pergeseran bit.

4 Jelaskan kenapa RAM bersifat volatile sedangkan Read Only Memory (ROM) tidak?

Perbedaan sifat volatile/non-volatile antara RAM dan ROM disebabkan oleh teknologi penyimpanan fisik yang berbeda secara fundamental.


RAM (Random Access Memory) — Volatile: RAM menggunakan kapasitor kecil (pada DRAM) atau flip-flop transistor (pada SRAM) untuk menyimpan setiap bit data. Kapasitor membutuhkan muatan listrik yang terus diperbarui (refresh) untuk mempertahankan nilai bit-nya. Ketika aliran listrik terputus, kapasitor kehilangan muatannya dalam hitungan milidetik, sehingga seluruh data hilang seketika. Inilah mengapa RAM bersifat volatile.

"DRAM stores data as electric charge on a capacitor. Since capacitors have a natural tendency to discharge, DRAM must be refreshed thousands of times per second to maintain its data." — Tanenbaum, A.S., Structured Computer Organization, 6th Ed., 2013. → Referensi: Tanenbaum, A.S. (2013). Structured Computer Organization, 6th Ed. Pearson.

ROM (Read Only Memory) — Non-Volatile: ROM menggunakan metode penyimpanan yang tidak bergantung pada aliran listrik. Pada mask ROM tradisional, data "dibakar" secara permanen ke dalam struktur transistor chip selama proses pembuatan (fabrikasi). Pada Flash ROM/EEPROM, data disimpan dalam floating-gate transistor yang mampu mempertahankan muatan elektron tanpa power selama bertahun-tahun, karena elektron terjebak dalam lapisan isolator yang mengelilingi gate.


Dengan kata lain: RAM menyimpan informasi melalui kehadiran arus listrik aktif, sedangkan ROM menyimpan informasi melalui struktur fisik material semikonduktor yang tidak memerlukan arus untuk mempertahankan data.

→ Referensi: Patterson, D.A. & Hennessy, J.L. (2017). Computer Organization and Design, 5th Ed., Appendix B. Morgan Kaufmann.
5 Jelaskan apa yang dimaksud operasi komputer? (sertakan gambar)

Operasi komputer adalah serangkaian langkah dasar yang dilakukan CPU secara berulang untuk mengeksekusi setiap instruksi program. Siklus ini dikenal sebagai Instruction Cycle atau Fetch-Decode-Execute Cycle.

1. Fetch (Pengambilan): CPU membaca instruksi dari alamat memori yang ditunjuk oleh Program Counter (PC), lalu menyimpannya di Instruction Register (IR). PC kemudian diinkrementasi untuk menunjuk instruksi berikutnya.


2. Decode (Penerjemahan): Control Unit menganalisis kode instruksi di IR untuk menentukan operasi apa yang harus dilakukan dan operand mana yang diperlukan.


3. Fetch Operand: Jika instruksi membutuhkan data dari memori, CPU mengambil data tersebut.


4. Execute (Eksekusi): ALU atau unit lain melaksanakan operasi yang ditentukan instruksi — bisa berupa operasi aritmatika, transfer data, atau perubahan alur program.


5. Store Result: Hasil operasi disimpan kembali ke register atau memori.

"The basic function performed by a computer is execution of a program, which consists of a set of instructions stored in memory. The processor does the actual work by executing instructions specified in the program." — Stallings, W., Computer Organization and Architecture, 10th Ed., 2016. → Referensi: Stallings, W. (2016). Computer Organization and Architecture, 10th Ed., Ch.3. Pearson.
6 Apakah data Adobe Illustrator yang belum disimpan akan hilang di cache memory saat terjadi pemadaman listrik?

Ya, data yang belum tersimpan tersebut akan hilang. Ini adalah konsekuensi langsung dari sifat volatile pada semua jenis memori yang digunakan untuk menyimpan data aktif selama program berjalan.


Mengapa data hilang?


Ketika kita membuat vektor di Adobe Illustrator, data gambar tersebut disimpan di RAM (Random Access Memory). Cache memory (L1/L2/L3 cache di dalam CPU) hanya menyimpan salinan sementara dari data RAM yang paling sering diakses, untuk mempercepat akses CPU. Keduanya — RAM dan Cache — bersifat volatile.

"Cache memory is a small, fast memory that holds copies of the data from the most frequently used main memory locations. Cache is also volatile, meaning its contents are lost when power is removed." — Stallings, W., Computer Organization and Architecture, 10th Ed., 2016. → Referensi: Stallings, W. (2016). Computer Organization and Architecture, 10th Ed., Ch.4. Pearson.

Ketika listrik padam secara tiba-tiba:


① Cache memory langsung kehilangan seluruh isinya karena tidak ada daya untuk refresh kapasitor.
② RAM pun sama — seluruh data gambar yang sedang dikerjakan langsung hilang.
③ Data hanya akan selamat jika sebelumnya sudah di-save ke storage non-volatile (HDD/SSD).


Catatan penting: Beberapa aplikasi modern (termasuk Adobe Illustrator) memiliki fitur Auto-Recovery yang secara berkala menyimpan file pemulihan ke disk. Namun, jika fitur ini tidak aktif atau interval autosave-nya belum tercapai, data tetap akan hilang.

→ Referensi: Patterson, D.A. & Hennessy, J.L. (2017). Computer Organization and Design RISC-V Ed., Ch.5 Memory Hierarchy. Morgan Kaufmann.
7 Jelaskan fungsi modul I/O untuk mensinkronkan kerja masing-masing komponen komputer!

Salah satu tantangan terbesar dalam arsitektur komputer adalah bahwa setiap komponen bekerja dengan kecepatan yang sangat berbeda-beda. CPU bekerja dalam orde GHz (milyaran siklus per detik), RAM dalam orde puluhan MHz, sedangkan perangkat I/O seperti HDD atau jaringan bekerja jauh lebih lambat. Modul I/O hadir sebagai manajer sinkronisasi yang menjembatani perbedaan ini.


Mekanisme sinkronisasi yang digunakan modul I/O:


1. Buffering: Modul I/O menyediakan area penyangga (buffer) yang menyimpan data sementara saat terjadi ketidaksesuaian kecepatan antara CPU dan perangkat. Saat CPU mengirim data ke printer, buffer menampung data tersebut sehingga CPU bisa langsung melanjutkan tugas lain tanpa menunggu printer selesai mencetak.


2. Interrupt Mechanism: Daripada CPU terus-menerus memeriksa status perangkat I/O (polling yang boros waktu), modul I/O mengirimkan sinyal interrupt ke CPU ketika perangkat selesai melakukan tugasnya. CPU bisa mengerjakan hal lain dan bereaksi hanya saat diinterupsi.

"The I/O module can relieve the processor of the burden of checking on I/O status through the use of interrupts. The processor can continue executing instructions while the I/O module handles the I/O operation." — Stallings, W., Computer Organization and Architecture, 10th Ed., Ch.7. → Referensi: Stallings, W. (2016). Computer Organization and Architecture, 10th Ed., Ch.7. Pearson.

3. DMA (Direct Memory Access): Untuk transfer data besar (mis. membaca file dari disk), modul I/O dapat mengambil kendali System Bus dan mentransfer data langsung antara perangkat dan memori tanpa melibatkan CPU untuk setiap byte, kemudian menginterupsi CPU hanya setelah transfer selesai.


4. Handshaking: Modul I/O mengimplementasikan protokol握手 (handshaking) antara perangkat dan sistem, memastikan kedua pihak siap sebelum transfer data dimulai, mencegah data hilang atau corrupt akibat timing yang tidak tepat.


Tanpa modul I/O sebagai sinkronisator, CPU harus menangani semua perbedaan kecepatan dan protokol secara langsung, yang akan sangat menurunkan performa dan kompleksitas sistem secara keseluruhan.

→ Referensi: Tanenbaum, A.S. (2013). Structured Computer Organization, 6th Ed., Ch.3. Pearson. | Stallings, W. (2016). Computer Organization and Architecture, 10th Ed., Ch.7.
